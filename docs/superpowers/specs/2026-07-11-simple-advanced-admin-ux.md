# 호갱 API 간편·상세 관리자 UX

## 목표

- 처음 사용하는 사람도 방송 준비 순서를 별도 설명 없이 이해한다.
- 방송 중에는 연결, 판매, 오류, 종료 상태를 한 화면에서 빠르게 조작한다.
- 세부 설정은 간편 화면에서 분리해 인지 부담을 줄인다.
- 기존 서버 API와 이치방쿠지 데이터는 그대로 유지한다.

## 참고 원칙

빵떡이 관리 화면의 시각 표현을 복제하지 않고 다음 사용성 원칙을 적용한다.

- 메뉴를 목적별 그룹으로 분리한다.
- 화면 상단에 현재 상태와 짧은 안내를 제공한다.
- 설정은 `항목명 · 설명 · 조작부` 순서의 행으로 통일한다.
- 자주 쓰는 운영 기능과 최초·저빈도 설정을 분리한다.
- 위험한 동작은 일반 저장 버튼과 색상·위치를 분리한다.

## 전체 구조

데스크톱은 기존 왼쪽 사이드바를 유지하고 다음 메뉴를 제공한다.

### 방송 운영

- `간편 운영`: 기본 진입 화면
- `판매 번호판`: 현재 회차 번호와 구매자
- `당첨 내역`: 회차별 당첨자 목록

### 설정

- `회차 설정`: 새 회차의 가격, 상품, 등급, 번호 구성
- `오버레이`: OBS 주소와 닉네임 표시
- `기타 설정`: 저빈도 기능

모바일은 하단에 `운영`, `번호판`, `당첨`, `더보기` 네 항목만 표시한다. `더보기`를 누르면 회차 설정, 오버레이, 기타 설정, 사용법, 로그아웃을 보여주는 바텀 시트를 연다.

## 간편 운영

첫 화면은 방송 준비와 운영 순서를 위에서 아래로 읽도록 구성한다.

1. **방송 연결**: 치지직 연결 상태와 재인증 동작
2. **이치방쿠지 상태**: 기능 토글과 현재 동작 여부
3. **현재 회차**: 회차명, 장당 가격, 판매 수, 남은 수
4. **처리 필요**: 문제가 있는 후원만 표시하고 각 행에서 처리 완료
5. **방송 마무리**: 회차 종료 버튼

진행 중인 회차가 없으면 3번 영역에 큰 `새 회차 만들기` 버튼을 표시하고 `회차 설정`으로 이동한다. 회차가 있으면 생성 폼을 간편 화면에 중복 표시하지 않는다.

화면 상단에는 `방송 준비 완료`, `확인 필요`, `회차 없음` 중 하나의 요약 상태를 표시한다. 상태는 연결, 기능 토글, 활성 회차, 처리 큐를 조합해 계산한다.

## 판매 번호판

- 번호판이 화면의 주 콘텐츠가 된다.
- 상단에 회차명, 판매 수, 남은 수, 장당 가격을 한 줄로 표시한다.
- `전체`, `판매 가능`, `판매 완료` 필터를 제공한다.
- 판매된 번호는 구매자를 함께 표시하고, 긴 닉네임은 말줄임과 툴팁을 사용한다.
- 회차 종료는 번호판 화면에 두지 않고 간편 운영의 방송 마무리 영역에만 둔다.

## 회차 설정

회차 생성은 세 단계로 나눈다.

1. 기본 정보: 회차 이름, 장당 가격
2. 상품 구성: 등급, 상품명, 수량
3. 번호 확인: 무작위 배치 결과 확인과 회차 시작

현재의 직접 번호 입력 방식은 `직접 편집` 상세 영역으로 유지한다. 기본 흐름은 등급별 상품과 수량을 입력하면 번호가 자동 생성되는 방식이다.

각 단계는 한 화면 안의 세로 섹션으로 유지하며 별도 페이지 전환은 하지 않는다. 필수값 누락 시 해당 행 바로 아래에 오류를 표시하고 첫 오류로 포커스를 이동한다.

## 당첨 내역

- 회차별로 묶어 최신 회차부터 표시한다.
- 검색 입력으로 닉네임, 상품명, 번호를 찾을 수 있다.
- 결과가 없으면 대기 마스코트와 짧은 빈 상태만 표시한다.
- 날짜는 한국 시간 기준의 읽기 쉬운 형식으로 표시한다.

## 오버레이

설정 행 두 개로 구성한다.

- `OBS 브라우저 소스`: 주소, 복사 버튼, 새 창 미리보기
- `닉네임 표시`: 부분 마스킹/전체 노출 선택과 개인정보 주의 설명

복사 성공은 버튼 문구와 작은 상태 메시지로 확인한다. 주소는 모바일에서 가로 스크롤 대신 여러 줄로 표시한다.

## 기타 설정

- 서비스 사용법 링크
- 로그아웃
- 향후 추가되는 저빈도 옵션

치지직 연결 재인증은 기타 설정으로 숨기지 않고 간편 운영의 연결 영역에서 제공한다.

## 피드백과 안전장치

- 저장·토글·처리 완료 작업은 진행 중 상태에서 버튼을 비활성화한다.
- 성공은 초록색 인라인 메시지, 실패는 코랄색 인라인 메시지로 표시한다.
- 회차 종료는 확인 대화상자에서 회차명과 종료 후 되돌릴 수 없음을 알린다.
- API `401` 응답은 로그인 화면으로 이동한다.
- API 실패 시 화면의 기존 데이터를 지우지 않고 재시도 버튼을 제공한다.

## 컴포넌트 경계

- `AppShell`: 데스크톱 사이드바, 모바일 하단 탐색, 더보기 시트
- `OperationsPage`: 준비 상태 계산과 간편 운영 섹션 조합
- `ConnectionRow`: 연결 상태와 재인증
- `FeatureRow`: 이치방쿠지 토글
- `CurrentSessionSummary`: 현재 회차 요약과 설정 이동
- `AttentionQueue`: 처리 필요 목록
- `TicketBoardPage`: 필터 가능한 번호판
- `SessionSetupPage`: 회차 생성 3단계
- `WinnersPage`: 검색과 회차별 당첨 목록
- `OverlaySettingsPage`: OBS 주소와 닉네임 설정
- `MoreSheet`: 모바일 상세 메뉴, 사용법, 로그아웃
- `ConfirmDialog`: 위험 동작 확인
- `InlineFeedback`: 진행·성공·실패 피드백

`App.tsx`는 인증 확인, API 초기 로딩, Socket.IO 구독과 최상위 페이지 전환만 담당한다.

## 반응형·접근성

- 데스크톱 기준 너비는 `1280x800`, 모바일 기준은 `390x844`다.
- 버튼 최소 높이는 `40px`, 모바일 주요 동작은 `44px`다.
- 상태는 색상만으로 전달하지 않고 아이콘과 문구를 함께 사용한다.
- 모든 입력에 연결된 라벨과 오류 문구를 제공한다.
- 모달과 바텀 시트는 열릴 때 포커스를 이동하고 닫을 때 원래 버튼으로 복귀한다.
- `prefers-reduced-motion`에서는 시트와 상태 애니메이션을 제거한다.

## 테스트와 검증

- 준비 상태 계산을 단위 테스트한다.
- 번호판 필터, 당첨 검색, 회차 설정 유효성 검사를 단위 테스트한다.
- 토글, 큐 처리, 회차 생성·종료의 API 호출과 오류 피드백을 컴포넌트 테스트한다.
- 기존 서버 테스트와 프로덕션 빌드를 모두 실행한다.
- Playwright로 데스크톱과 모바일의 간편 운영, 번호판, 회차 설정, 오버레이 화면을 캡처한다.
- 가로 넘침, 겹침, 잘린 한국어 문구, 키보드 포커스, 모바일 더보기 시트를 확인한다.

## 범위 제외

- 서버 데이터 모델 변경
- 결제 시스템
- 여러 채널 동시 관리
- 권한별 메뉴 제한
- 빵떡이의 색상, 로고, 캐릭터 또는 화면을 그대로 복제하는 작업
