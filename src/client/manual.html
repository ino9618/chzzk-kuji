호갱 API 사용법

치지직 후원과 연동해 번호별 상품을 자동으로 배정하는 프로그램입니다. 최초 설정이 끝났다면, 방송할 때마다 아래 순서대로 진행하세요.

1. 프로그램 켜기

Web/chzzk-kuji 폴더에서 npm run dev 실행. 로컬 PC에서 쓰는 경우 이 창을 방송 내내 켜두어야 후원을 계속 받을 수 있습니다. 껐다 켤 때마다 치지직 계정을 다시 인가할 필요는 없습니다 — 연결 정보는 이미 저장되어 있습니다.

2. 회차 만들기

  1. 관리자 화면(/admin.html) 접속 → 네이버 계정으로 로그인 버튼 클릭. 모든 네이버·치지직 계정이 별도 초대 없이 관리자 화면에 로그인할 수 있습니다. 가장 처음 로그인한 계정은 후원 수신 채널의 소유자로 등록되고, 소유자 로그인과 동시에 치지직 후원 수신 연결도 자동으로 시작됩니다.
  2. 간편 운영에서 방송 연결이 연결됨인지 확인합니다.
  3. 회차 설정으로 이동해 기본 정보와 상품 구성을 입력합니다:
    • 회차 이름: 자유롭게 (예: "7월 6일 이치방쿠지")
    • 장당 가격: 티켓 1장당 치즈 금액 (예: 1000)
    • 최대 번호: 몇 번까지 팔 것인지 (예: 10이면 1~10번)
    • 번호,상품명,등급(선택): 한 줄에 하나씩, 쉼표로 구분 (예: 1,A상)
  4. 번호 확인에서 "회차 시작" 클릭 → 판매 번호판이 뜨면 준비 완료
모바일에서는 하단의 더보기에서 회차 설정과 오버레이를 열 수 있습니다. 방송 중에는 운영 화면만 확인하면 연결, 판매 현황, 처리 필요 항목, 회차 종료를 한 번에 관리할 수 있습니다.

3. 후원 받기

시청자가 "장당 가격"만큼 치즈를 후원하면서 메시지에 원하는 번호(예: "3번")를 적으면 자동으로 처리됩니다.

번호가 아직 안 팔렸으면 → 즉시 그 시청자에게 배정되어 번호판/오버레이에 반영됩니다.
이미 팔린 번호거나, 금액이 안 맞거나, 번호를 안 적었으면 → 배정되지 않고 "처리 필요 큐"에 기록만 됩니다. 환불은 자동으로 되지 않으니 방송에서 직접 안내해주세요.

4. OBS에 오버레이 넣기 (한 번만 설정)

OBS에서 소스 추가 → 브라우저 선택 → 쿠지 번호판은 http://localhost:5173/overlay-kuji-board.html, 당첨 애니메이션은 http://localhost:5173/overlay-kuji-result.html, 룰렛은 http://localhost:5173/overlay-roulette.html 입력. 배경이 투명한 번호판이 화면에 뜹니다. 판매된 번호는 초록색으로 강조되고, 방금 팔린 번호는 잠깐 반짝이는 효과가 나타납니다.

5. 회차 마무리

번호가 다 팔렸거나 방송을 마무리할 때 관리자 화면에서 "회차 종료" 클릭. 종료된 회차 기록은 삭제되지 않고 남아있고, 다음 방송에는 "회차 시작"으로 새로 만들면 됩니다.

6. 처리 필요 큐 확인

"처리 필요 큐"에 쌓인 항목(중복 후원, 금액 불일치 등)은 방송 중/후에 확인해서 시청자에게 안내하거나 필요하면 직접 처리한 뒤, "처리완료" 버튼으로 정리하세요.

문제 생겼을 때

연결 상태가 "reconnecting"

일시적 문제입니다. 잠시 후 자동으로 connected로 돌아옵니다.

빨간 배너("재인증 필요")가 뜰 때

배너의 "네이버로 다시 로그인" 링크를 클릭해 인가를 완료하면 후원 수신 연결이 즉시 자동으로 복구됩니다. 서버를 재시작할 필요는 없습니다.

로그인과 관리자 계정

접속은 네이버 계정 로그인만 지원합니다. 처음 로그인한 치지직 계정이 소유자로 등록됩니다. 그 밖의 모든 정상 네이버·치지직 계정도 별도 초대 없이 관리 화면을 사용할 수 있습니다. 소유자가 아닌 계정의 로그인은 후원 수신 연결에는 영향을 주지 않습니다(소유자 계정만 치지직 소켓을 관리). 로그아웃은 왼쪽 메뉴 하단의 로그아웃 버튼을 쓰세요.

치지직 OAuth 장애 등으로 로그인이 아예 불가능해질 때를 대비한 비상용 접근 방법은 프로젝트 README에 정리되어 있습니다.
